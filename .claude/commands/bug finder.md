## 報告請包含以下幾項

### 1. 真 bug（會影響實際行為的錯誤）
列出會導致玩家體驗異常、計算錯誤、或執行出錯的問題。每項註明：在哪個檔案哪幾行、會造成什麼後果、嚴重程度（高/中/低）。

### 2. 廢碼 / 死碼（沒有作用的殘留）
列出已經沒被任何地方使用、或邏輯上不可能執行到的程式碼。**注意區分**：
- 真正的死碼（沒接上、純殘留）→ 可清除
- **「刻意的設計」**（例如氛圍用的假判定：成敗指向同一節點）→ **不是廢碼，不要建議移除**。如果 code 裡有註解標明是刻意的，請尊重該註解。

### 3. 違反 CLAUDE.md 規範之處
檢查有沒有違反 CLAUDE.md 立下的原則——**最重要的一條：副本專屬邏輯一律寫進 WINMODE_HOOKS，絕不寫進引擎主流程（如 choose()）**。列出任何把副本特例硬寫進核心的地方。

### 4. 特例風險（技術債來源）

### 5. 數值散落
檢查 DC、reward、掉落率、加成等數值是寫死在各副本各處，還是有集中管理。如果散落，指出來。

### 6. 修改風險點
以目前結構，哪些地方「改一處會壞多處」？列出最危險的幾個共用邏輯。
