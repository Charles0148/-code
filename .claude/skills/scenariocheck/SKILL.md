# /scenariocheck — 劇本結構稽核

當作者輸入 `/scenariocheck` 時：

1. 於 repo 根目錄執行：`node tools/scenario_audit.js`
2. 擷取腳本的完整 stdout 與退出碼。
3. 呈現結果：
   - 退出碼 0 → 先回報「✅ 劇本結構稽核通過，無幽靈節點、契約機制誤用、成就幽靈、win 型別非法或幽靈旗標」，再附上各劇本摘要。
   - 退出碼 1 → 先回報「⚠ 發現 N 個劇本結構問題」，把「問題清單」段落放最前面，再附各劇本摘要供對照。
4. 孤兒節點（ⓘ）僅供參考、不計入退出碼，不要當成需修復的問題；僅靠 turnLimit 強制結束到達的 end 節點出現在孤兒清單屬正常。
5. 僅回報，不自動修改任何檔案。任何修正方案都交由作者決定後另行處理。
